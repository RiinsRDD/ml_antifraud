Transaction Processing Flow

1

Transaction Ingestion

Client sends transaction data via POST request to /api/v1/transactions endpoint. FastAPI validates data using Pydantic models.

2

Data Processing

TransactionService processes the request, calculates customer statistics from Redis cache, and prepares features for scoring.

3

ML Model Scoring

ScoringService calls the ML model to evaluate fraud risk. Implements timeout handling with fallback scoring mechanism.

4

Result Storage

Transaction data and scoring results are stored in Redis cache for future reference and statistics calculation.

5

Response Generation

System returns scoring result with fraud probability, processing time, and customer statistics to the client.

6

Monitoring & Metrics

Prometheus metrics are updated with request count, latency, and system performance data for dashboard visualization.
